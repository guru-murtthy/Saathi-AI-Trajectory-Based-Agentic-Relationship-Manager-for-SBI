# Policy Mapping & Compliance Matrix

This document maps the architectural components of the **Saathi AI** platform to the regulatory acts and guidelines set by the Government of India and the Reserve Bank of India (RBI).

---

## 1. Digital Personal Data Protection (DPDP) Act 2023

### A. Consent Capture & Management
*   **Regulatory Requirement**: Data processing must be backed by an active, purpose-scoped, and revocable consent.
*   **Implementation**: Enforced by [dpdp.py](file:///home/gururaj/Videos/sbi/sbi-main/backend/app/compliance/dpdp.py) via the `ConsentStore` service and `ConsentRecord` model.
*   **Enforcement Point**: Before generating a recommendation, `rm.py`'s `advise()` method checks `ConsentStore.is_active(customer_id, "recommendation")`. If consent is absent or revoked, processing halts, and a fallback denied-access payload is returned.

### B. Purpose Limitation & Data Minimization
*   **Regulatory Requirement**: Personal data must be processed only for the specified purpose, and only the minimum required data should be accessed.
*   **Implementation**:
    *   **Purpose Restriction**: The consent gate restricts usage strictly to `"recommendation"`.
    *   **Data Minimization (ChromaDB)**: The peer-cohort engine ([community.py](file:///home/gururaj/Videos/sbi/sbi-main/backend/app/engines/community.py)) uses anonymous numerical feature embeddings in ChromaDB to search for similar cohorts. Raw Personally Identifiable Information (PII) like names, phone numbers, or account numbers is never synced or stored in vector search indices.

## 2. RBI Digital Lending Guidelines

### A. Human-in-the-Loop (HITL) for Financial Actions
*   **Regulatory Requirement**: Automated decisioning tools must not trigger automatic disbursals or transfers without human-in-the-loop validation and explicit sign-offs.
*   **Implementation**: Every recommendation generated by the RM agent has its `requires_human_approval` flag set to `True` (implemented in [rm.py](file:///home/gururaj/Videos/sbi/sbi-main/backend/app/agent/rm.py#L70)).
*   **Enforcement Point**: The frontend displays an alert confirming that the recommended action requires manual RM review and confirmation, preventing automated execution of loan applications.

---

## 3. RBI FREE-AI Framework

### A. Explainability & Interpretability
*   **Regulatory Requirement**: AI/ML models deployed in credit scoring, profiling, or product recommendations must be explainable (no opaque "black-box" decisions).
*   **Implementation**: Satisfied by the **Life Event Knowledge Graph** ([life_event_graph.py](file:///home/gururaj/Videos/sbi/sbi-main/backend/app/engines/life_event_graph.py)).
*   **Explanation Artifact**: The reasoning loop constructs a traceable path: `Observed Signal` $\rightarrow$ `Predicted Event` $\rightarrow$ `Financial Need` $\rightarrow$ `Recommended Journey`. The RM narrative relies on this structured graph's predecessor-successor links to assemble human-readable reasoning, showing precisely which transactions triggered the recommendation.

---

## 4. Data Retention Policy

### A. Retention Limits
*   **Policy Rule**:
    *   **Raw Transaction Features**: Retained for exactly **90 days** to compute real-time trajectories.
    *   **Anonymized Aggregates / Cohort Embeddings**: Retained indefinitely for model retraining and benchmarking.
*   **Enforcement Point**: In production, a database retention scheduler deletes transactions older than 90 days. In the application code, the transaction filter in [generator.py](file:///home/gururaj/Videos/sbi/sbi-main/backend/app/data/generator.py) restricts transaction simulation to a rolling window, and features are computed only on the relevant active transaction trails.
